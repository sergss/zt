# Инструкции по разработке проекта

## Окружение и ограничения
- **No Node.js / No Frameworks**: Проект реализован на чистом (native) JavaScript. Запрещено использовать библиотеки, фреймворки или устанавливать зависимости.
- **No Build Tools**: В проекте отсутствуют команды для сборки (npm, webpack, vite и т.д.). Приложение запускается напрямую через браузер.
- **Запуск**: Приложение открывается открытием файла `index.html`.
- **TypeScript**: В проекте НЕ используется TypeScript для разработки. Не нужно пересобирать проект с помощью команды `npx tsc --noEmit`.

## Структура проекта
- **Логика**: Весь код игры находится в папке `./js/`. Файлы подключаются через `<script>` в `index.html`.
- **Тестирование**: Все тесты расположены в папке `./tests/`.
- **Запуск тестов**: Тесты запускаются автоматически при открытии файла `test-runner.html` в браузере.

## Требования к коду
- Пиши только нативном JS, совместимом с современными браузерами.
- Не предлагай установку пакетов или использование `npm install`.

## Окружение терминала (Terminal Environment)
- **OS**: Windows 11.
- **Shell**: Windows PowerShell 5.1 (Desktop Edition).
- **Execution Policy**: Учитывайте ограничения на запуск скриптов .ps1.
- **Command Syntax**: 
  - Используйте синтаксис PowerShell 5.1.
  - Разделитель команд — `;`.
  - Пути используйте только с обратным слэшем: `.\js\script.js`.
  - Для вывода файлов используйте `Get-Content`, для списка — `Get-ChildItem`.

## Языковые правила и работа с комментариями
- **Язык общения**: Ответы всегда должны быть на русском языке.
- **Язык кода**: Комментарии внутри JS-файлов должны быть только на русском языке.
- **Сохранение контекста**: Нельзя удалять существующие комментарии в коде, если изменения не вносятся в соответствующий блок кода.
- **Актуализация**: При изменении логики функции или блока кода, необходимо обновить или создать комментарии на русском языке, чтобы они соответствовали новому коду.
- **Формат**: Для описания функций и логики в папке `js/` использовать стандартные JSDoc комментарии.
